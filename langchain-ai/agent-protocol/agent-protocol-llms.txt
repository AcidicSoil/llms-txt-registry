# Agent Protocol

> The Agent Protocol aims to standardize the APIs required to serve LLM agents in production environments, enabling interoperability across different platforms and frameworks. It provides a unified interface for executing agent runs, managing multi-turn conversations with persistent state, introspecting agent capabilities, handling background tasks, and storing long-term memory. The protocol is designed to be framework-agnostic and extensible, with support for various use cases including chatbots, research agents, and stateless interactions. It also encourages community implementations, such as LangGraph.js and the LangGraph Platform, to build upon this foundation.

**Remember:**
- Runs: APIs for executing agent tasks, including stateless and background modes
- Threads: Persistent, multi-turn conversations with state history and concurrency controls
- Store: Long-term memory management with customizable scopes and structured data support
- Agents: Introspection endpoints to discover agent capabilities, input/output schemas, and metadata
- Messages: Standardized message format for LLM interactions, supporting streaming and structured output
- Background Runs: Non-blocking agent execution with support for cancellation, streaming, and retries
