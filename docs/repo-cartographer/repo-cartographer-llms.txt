# Repo Cartographer

> Repo Cartographer aims to create a flexible, AI-driven development environment where users can generate, plan, and scaffold software repositories using a combination of DSPy-based reasoning and a structured planning graph (RPG). It separates user-facing interfaces—CLI and TUI—from the core logic, enabling independent evolution and improvement of each layer. The project centralizes configuration and model provider management to allow seamless switching between different language models, while introducing an RPG layer that validates repository structure, plans feature hierarchies, and generates deterministic build and test queues. Ultimately, the goal is to build a scalable, extensible, and maintainable system that supports intelligent code generation, automated repository planning, and rich user interaction through both command-line and graphical interfaces.

**Remember:**
- CLI agent structure
- TUI (Textual User Interface)
- DSPy modules and signatures
- Repository Planning Graph (RPG)
- Provider registry and environment configuration
- Prompt templates and program instructions
