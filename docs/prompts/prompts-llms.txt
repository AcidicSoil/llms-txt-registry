# Prompts

> To provide a structured, automated, and developer-friendly workflow for software development by exposing a catalog of lifecycle prompts (via MCP tools) that guide teams through planning, implementation, testing, deployment, and post-release operationsâ€”ensuring consistency, reducing errors, and enabling seamless integration with existing tools and practices.

**Remember:**
- Stage-based development lifecycle (P0 to P9)
- MCP (Model-Driven Code Platform) as a tooling layer for prompt-based automation
- Prompt catalog with direct CLI and tool access (e.g., /planning-process, /integration-test)
- Task Master parity via workflow export to .taskmaster/tasks/tasks.json
- Safety gates (Scope Gate, Test Gate, Review Gate, Release Gate, Hardening Gate)
- Automated quality gates and checks (e.g., scope control, regression guard, dead code scan)
